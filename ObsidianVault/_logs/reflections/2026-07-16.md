# Reflection - 2026-07-16

## Discord authentication delivery

### What went well

- Reused the existing single-use external exchange and ASP.NET Identity login model without a schema migration.
- Kept Google URLs and behavior while binding protected callbacks and link intents to an allowlisted provider.
- Added deterministic Discord coverage for registration, login, conflicts, linking, untrusted identities, dual-provider accounts, and disconnection.

### Risks kept explicit

- Real Discord activation remains configuration-dependent and requires owner-created credentials.
- Discord-only accounts depend on the provider remaining enabled unless the user adds another sign-in method.
- Discord guild membership, bots, avatars, and retained OAuth tokens remain outside this authentication feature.
