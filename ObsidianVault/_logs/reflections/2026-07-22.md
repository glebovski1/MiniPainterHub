# Reflection - 2026-07-22

## Database load optimization delivery

### What went well

- Reduced entity-graph materialization on project search, project paging, message send, and post-details bootstrap while preserving DTOs and compatibility endpoints.
- Added measurements before relying on infrastructure scaling, and kept SQL text and parameter values out of logs.
- Limited caching to short-lived anonymous reads using the framework's authentication and response-safety rules.

### Risks kept explicit

- Substring search remains non-sargable without a future full-text-search decision.
- Output caching is intentionally process-local and short-lived; multi-instance cache coherence is not assumed.
- Index value must be confirmed with production query telemetry after the migration is deployed.

### Follow-up trigger

- Reassess full-text search or the Azure SQL tier only if measured request command counts are healthy but database duration remains above the configured threshold.
