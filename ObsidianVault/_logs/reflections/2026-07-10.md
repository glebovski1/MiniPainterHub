# Reflections - 2026-07-10

## 14:50 UTC — Reflection on MiniPainterHub brand identity addition

### What went well

- The logo system reuses the established product palette and typography direction instead of creating a parallel visual language.
- The asset set covers primary, reversed, compact, and review contexts without duplicating raster source files.
- Design rationale, accessibility guidance, minimum sizes, and the future runtime integration boundary are documented together.

### Possible bad patterns detected

- The editable wordmark depends on the product font stack and may render with a fallback when viewed outside the app or repository environment.

### Fixes applied now

- Added explicit fallback fonts to every SVG wordmark.
- Documented that fixed external exports should convert the final approved wordmark to outlines while keeping the repository source editable.
- Kept runtime navbar and favicon integration out of the design proposal so visual approval remains a separate review decision.

### Fixes recommended later

- After approval, create outlined distribution exports and run the full shared-shell UI review when integrating the mark into navigation and browser metadata.
