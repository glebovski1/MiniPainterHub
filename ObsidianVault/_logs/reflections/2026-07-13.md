# 2026-07-13

## Reflection on Hobby Projects delivery

### What went well

- Reused posts as the image, discussion, reaction, recipe, tag, and rich-viewing source of truth, so projects add durable organization without creating a second publishing stack.
- Kept ownership, public eligibility, completion, reopening, archive, moderation, cover fallback, and cross-project move rules in one service boundary, with SQL Server coverage for uniqueness and duplicate-link races.
- Exercised the whole painter journey in one browser scenario and repeated the full visual sweep after inspecting every generated desktop and mobile capture.
- Used visual evidence to uncover and fix issues beyond the new screens, including hidden toasts, unreadable Bootstrap compatibility badges, disabled-button palette leakage, mobile viewer clipping, and a real pointer-hit bug on the author-note composer.

### Possible bad patterns detected

- Authorization and restriction checks initially ran in an order that could reveal a same-resource restriction before applying the required cross-owner `404` boundary.
- Project-aware cached post/search projections could retain a public project reference after visibility changed.
- Early screenshot fixtures leaked local drafts between groups, captured a loading spinner as an empty state, and depended on cached image-request timing.
- Component tests could invoke the author-note cancel callback even while browser CSS made the composer non-interactive.

### Fixes applied now

- Applied ownership isolation before same-owner restriction checks and added focused controller/service boundary tests.
- Removed unsafe caching from visibility-sensitive project projections and added real transition tests.
- Cleared client fixture state, asserted meaningful settled states, bounded the retry-image fixture, and stabilized profile copy before capture.
- Restored composer pointer hit testing, explicitly invoked cancel, isolated its events from the viewer stage, and covered both component and real-browser dismissal.
- Added Bootstrap 5.1 compatibility rules and regressions for light badges plus primary active, focus, and disabled states.
