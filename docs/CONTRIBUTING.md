# Contributing

## Getting Started
- Ensure the solution builds locally with `dotnet build`.
- Keep changes focused and aligned with the existing architecture.

## Pull Requests
- Describe the problem and solution clearly.
- Note any tradeoffs, risks, or follow-up work.

## Code Expectations
- Follow the conventions in `docs/CODE_STYLE.md`.
- Avoid anti-patterns listed in `docs/ANTI_PATTERNS.md`.
- Use the service layer for business logic.
- If AI assisted a significant change, add an entry to `docs/AI_CHANGELOG.md`.
